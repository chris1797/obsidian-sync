로그인은 기본적으로 인증([[Authentication]]), 인가([[Authorization]]) 라는 개념이 적용됨.

1. 클라이언트는 API 요청 시 "Authentication" Header에 JWT 토큰을 포함하여 서버에 전송함.
2. 서버의 Security Filter가 해당 요청을 가로채서 JWT 토큰의 유효성을 검증함.
3. 유효하면 SecurityContextHolder에 사용자 인증 정보를 저장한 후 컨트롤러로 요청을 전달.
4. 컨트롤러에서 @PreAuthorize, @RolesAllowed 등 어노테이션으로 권한을 검증하여 접근 제어.
