# SQL 튜닝 MOC

이 영역의 입구. 핵심 축은 **"DB는 어느 메모리에서 일하느냐"** — 조인 알고리즘의 차이도 결국 PGA/SGA 선택 전략이다.

## 읽는 순서

1. [[PGA]] / [[SGA]] — 먼저 짝으로. 공유/사적·래치 유무가 다음 챕터의 토대.
2. [[NL 조인]] — SGA에서 일함. 소량+인덱스에 강함, 대량엔 래치 누적으로 약함.
3. [[해시 조인]] — PGA에 Build 올려놓고 O(1) 매칭. NL의 SGA 한계를 우회한 전략.
4. [[소트 머지 조인]] — 정렬 후 머지. 해시가 안 될 때의 대안.
