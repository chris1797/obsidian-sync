# Obsidian Vault — 작업 지침

이 디렉토리는 **개발/CS/DB 개념을 연결하며 이해하기 위한 개인 지식 볼트**입니다. 코드 저장소가 아니라 Obsidian 노트 모음이므로, 일반 코드 프로젝트 규칙(테스트 작성·린트 등)은 적용하지 마세요.

## 볼트 성격

- 노트는 "지식의 산물" — 단편 암기가 아닌 **개념 간 연결**을 통한 깊은 이해가 목적.
- 톤은 **개인 학습 노트**: 교과서식 형식보다, 이해한 흐름·맥락·의문점이 드러나는 글.
- 관심 영역은 **프레임워크/DB의 내부 메커니즘**(JPA 동작 원리, Spring Security 흐름, SQL 옵티마이저·메모리 구조 등) — 단순 사용법보다 작동 원리.

## 폴더 구조

주제별 폴더 + 그 안에 개별 노트:

- `JPA/` — JPA/Hibernate 동작 원리
- `SpringSecurity/` — 인증/인가 흐름
- `SQL 튜닝/` — RDBMS 내부와 튜닝
- 루트 — 폴더에 묶이지 않는 일반 개념 노트

새 주제는 같은 패턴으로 폴더를 만들어 분류.

## 노트 작성 규칙

- **Obsidian `[[링크]]`로 관련 노트와 적극 연결**할 것. 예: JPA N+1 ↔ SQL 조인 전략, Spring Security 인증 ↔ 세션/JWT.
- 새 노트를 만들기 전에 기존 노트와 연결 가능한 지점을 먼저 살피고, 본문에 그 연결을 명시.
- 단순 정보 나열 대신 **"왜 이 개념이 다른 개념과 연결되는가"**를 드러내는 구조 우선.
- **존재하지 않는 노트로 `[[링크]]`를 걸었다면, 작업을 마치고 사용자에게 그 노트도 새로 작성할지 제안할 것.** 빈 링크가 쌓이면 그래프가 비어버리니, 링크를 만든 책임은 노트 작성 제안까지 이어진다. 단, 사용자 승인 전에 임의로 새 파일을 만들지는 말 것.

## 다루지 말 것

- `0. Index.base` — Obsidian Bases(쿼리/뷰) 파일. 일반 마크다운 편집기로 손대지 말 것.
- `.obsidian/` — Obsidian 설정 디렉토리. 사용자가 명시적으로 요청할 때만 수정.
